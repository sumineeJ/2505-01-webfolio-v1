🧾 인하대학교 홈페이지 리뉴얼 퍼블리싱
🎓 너무 많고 복잡한 서브페이지들 속에서 부딪히며 배운 실전 퍼블리싱
📍 프로젝트 개요
  • 대상: 인하대학교 홈페이지 리디자인
  • 목표: 전체 페이지를 퍼블리싱하며, 사용자 흐름을 고려한 구조 설계와 반응형 구현
🛠️ 퍼블리싱 작업
  • 약 30개의 서브페이지를 포함하는 대규모 퍼블리싱 프로젝트
  • 디자인과 퍼블리싱을 병행, 모든 페이지의 시안을 직접 제작하고 HTML/CSS로 구현
  • 구조 정리나 컴포넌트화보다도 “하나라도 더 완성하자”는 마음으로 밀어붙였던 프로젝트
  • 디자인/퍼블리싱 분리가 명확하지 않았던 만큼, 코드를 정리하기보단 그때그때 해결하며 진행
📱 반응형 구현
  • PC, 태블릿, 모바일 해상도 기준으로 기본적인 레이아웃 조정
  • 디테일한 UI 완성도는 아쉬운 부분이 있지만, 다양한 디바이스에 맞춰 테스트하고 수정
  • 특정 해상도에서 레이아웃이 깨질 때마다 직접 부딪히며 조정
⚠️ 미완성에 대한 처리 방식
  • 모든 서브페이지를 완성하지는 못했지만, 마우스 오버 시 “아직 작업 중입니다” 메시지로 안내
  • 완성도 면에선 아쉬움이 있었지만, '작업의 흐름과 우선순위 설정'을 배우는 계기
💭 회고

이 프로젝트는 “웹 퍼블리싱은 체력전이다”라는 말을 몸소 실감하게 해 줬어요.
처음에는 가볍게 시작했지만, 페이지 수와 구조, 디자인까지 전부 혼자 감당하면서
퍼블리싱의 난이도와 중요성, 그리고 우선순위 설정의 필요성을 깨닫게 됐어요.

코드도 구조도 완벽하지 않았지만, 끝까지 손을 놓지 않고 마무리를 고민했다는 점에서는 의미 있는 경험이었고,
특히 완성되지 않은 페이지에 대한 안내 처리를 통해 사용자에 대한 배려를 실현하려 했던 점이 제 나름의 소득이었어요.

한마디로, “내가 만든 만큼 성장했던 프로젝트”
다시 하라고 하면 자신은 없지만 정말 많이 배웠어요.
